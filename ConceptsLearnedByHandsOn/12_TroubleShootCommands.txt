                                       CONCEPT: SHELL and TERMINAL

Terminal = the UI (the window/program you type into)

It handles:
    displaying text
    taking your keyboard input
    showing output from programs

The terminal does not understand commands.
It just shows whatever the shell prints.

Shell = the agent (the program that interprets your commands)

It handles:
    parsing the commands you type
    running them
    managing environment variables
    scripting
    pipes, redirects, history

It is the brain basically, that understand, parse and outputs

When you open a terminal, it usually launches a shell automatically (bash, zsh, sh, fish, etc.).

Bash (Bourne Again shell) is one specific type of shell.
It is the most widely used shell on Linux and macOS (before macOS switched to zsh by default).

Terminal = the screen + keyboard interface
Shell = the program that reads and executes commands

You type into the terminal → the shell receives it → executes → terminal prints result.
